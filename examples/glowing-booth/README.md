# Glowing Booth — a worked example

This folder shows everything the **molino** engine reads when it builds work for a single brand. No API keys required to browse — you can read straight through and see how brand context flows into shipping HTML.

**Glowing Booth** is a real photo booth brand built on top of this engine. The brand DNA is high-contrast editorial: charcoal on vanilla, orange as the earned accent, generous display type, analog warmth.

---

## What's in this folder

```
glowing-booth/
├── figma-design-system.md   ← token source of truth (colors, type, spacing)
├── design-taste.md          ← aesthetic compass (what looks right)
├── design-director.md       ← brand strategy + voice (creative direction)
├── tokens/
│   ├── tokens.json          ← extracted from Figma via extract-tokens.js
│   └── variables.css        ← auto-generated CSS custom properties
└── output/
    └── index.html           ← a complete photobooth landing page generated by the engine
```

---

## How to read it

1. **Start with `figma-design-system.md`** — the literal token values. Every color, font, spacing unit Claude is allowed to use.
2. **Then `design-taste.md`** — *why* those tokens are arranged the way they are. The aesthetic logic.
3. **Then `design-director.md`** — *what* the brand is fighting for. Strategy, voice, campaign thinking.
4. **Then `output/index.html`** — open it in a browser. That's what the engine produces when given those three inputs plus a prompt like *"build a landing page for the photobooth product"*.

---

## What the engine actually did

Given those three context files, Claude:

1. Loaded the universal engine skills (`engine/skills/frontend-design.md` + `engine/skills/ui-animation.md`)
2. Loaded all three brand context files in the order above
3. Pulled token values from `tokens/variables.css` (every color, type size, and spacing var)
4. Generated a complete responsive HTML page using only the allowed tokens, the brand's typographic voice, and the editorial decisions documented in `design-taste.md`
5. Wrote it to `outputs/brand-a/websites/photobooth/index.html`

The output is vanilla HTML + CSS. No build step, no framework, no JS dependency beyond a CDN'd GSAP for the scroll choreography.

---

## Try it

```bash
# Open the generated page in your browser
open output/index.html
```

The page hero, type system, color use, and motion choices should all trace back to one of the three context files. That's the whole point — brand context goes in, on-brand HTML comes out.
